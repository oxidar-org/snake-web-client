# Snake Web Client

A browser-based multiplayer snake game client. The game logic runs in a Rust/WebAssembly module (`ferris`) compiled with [`wasm-pack`](https://rustwasm.github.io/wasm-pack/); a React/Vite app provides the thin JS host layer.

The client connects to the game server at `wss://snakes.hernan.rs` using a binary WebSocket + MessagePack protocol. See [CLIENT_SPEC.md](CLIENT_SPEC.md) for the full protocol reference.

---

## Requirements

### 1. Install Rust

Follow the official instructions at https://www.rust-lang.org/tools/install

```bash
rustc --version
```

### 2. Install wasm-pack

```bash
curl https://rustwasm.github.io/wasm-pack/installer/init.sh -sSf | sh
wasm-pack --version
```

### 3. Install Node.js and yarn

- Install Node.js from https://nodejs.org/ or via [nvm](https://github.com/nvm-sh/nvm)
- Enable yarn: `corepack enable`

```bash
node -v
yarn -v
```

## Project Structure

```
.
├── rust/               # Rust/WASM crate (crate name: ferris)
│   └── src/
│       ├── lib.rs      # Board, WebSocket, rendering
│       ├── protocol.rs # MessagePack encode/decode
│       └── error.rs    # Error types
├── js/                 # Vite + React app
│   └── src/
│       └── wasm/       # Generated by wasm-pack (do not edit)
├── build.sh            # Compiles Rust → WebAssembly into js/src/wasm/
├── CLIENT_SPEC.md      # WebSocket protocol and game rules
└── SERVER_SPEC.md      # Server configuration reference
```

---

## How to Run

1. Build the WASM module:

   ```bash
   sh build.sh
   ```

   This compiles `rust/` with `wasm-pack` and places the output in `js/src/wasm/`.

2. Start the dev server:

   ```bash
   cd js
   yarn
   yarn dev
   ```

   The app is served at `http://localhost:5173`.

---

## Deployment

The app deploys to GitHub Pages via the workflow in `.github/workflows/static.yml`. The Vite build uses the base path `/snake-web-client/` when the `GITHUB_PAGES` environment variable is set.
